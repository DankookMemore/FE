## MEMO-RE 플랫폼 소개

### 기존 메모의 문제점
- **흩어진 메모 기록**  
  “메모를 해도 여러 노트나 앱에 흩어져 있어, 한 번 기록한 아이디어를 다시 찾기 위해 여러 곳을 뒤져야 합니다.”
- **정리의 어려움**  
  “메모를 모아두어도 카테고리나 날짜별 정리가 미흡하여, 나중에 활용하기가 쉽지 않습니다.”
- **활용도 저하**  
  “필연적으로 흩어진 메모들이 제때 검토되지 못해, 유용한 아이디어가 무의미하게 사라지는 경우가 많습니다.”

### MEMO-RE가 제공하는 솔루션
- **통합된 워크스페이스 제공**  
  보드(폴더)별로 메모를 시각적으로 관리하여, 메모가 흩어지지 않고 체계적으로 분류됩니다.
- **아이디어 기록의 일관성 확보**  
  한 곳에 모든 메모를 모아두어, 떠오르는 순간 즉시 기록하고 나중에 한눈에 확인할 수 있습니다.
- **아이디어 재발견 촉진**  
  통합된 화면과 요약 기능으로 잊힌 영감을 다시 꺼내볼 수 있어, 기록된 아이디어가 온전히 활용됩니다.
- **AI 요약·분류 기능 도입**  
  ChatGPT를 활용해 작성된 메모를 자동으로 요약·분류해 주어, 이 메모가 어떤 아이디어와 연결되는지 쉽게 파악할 수 있습니다.
- **이웃(팔로우) 기능을 통한 아이디어 공유**  
  이웃을 팔로우하여 서로의 보드를 구독하고, 다른 사용자가 작성한 아이디어를 손쉽게 확인할 수 있습니다.

# 실행방법

## 프론트 실행 방법
1. git clone https://github.com/DankookMemore/FE.git
2. cd FE
3. npm install            # ← 이 단계로 필요한 패키지를 모두 설치합니다.
4. npx expo start         # Expo 개발 서버를 시작합니다.
5. w                      # 터미널에서 “w”를 누르면 웹 브라우저로 앱을 띄울 수 있습니다.

---

## 백엔드 실행 방법

1. 레포지토리 클론
git clone https://github.com/DankookMemore/BE.git

2. 클론한 디렉터리로 이동
cd BE

3. 가상환경 생성 및 활성화
python -m venv venv
Windows
venv\Scripts\activate

4. 백엔드 코드 디렉터리로 이동
cd DANKOOKMEMORE/memo_backend

5. 필요한 패키지 설치
pip install -r requirements.txt

6. 마이그레이션 수행
python manage.py makemigrations
python manage.py migrate

7. 개발 서버 실행
python manage.py runserver

## 화면별 기능 소개
### 1. 로그인 화면 (LoginScreen)
- **무엇을 하나요?**  
  이메일(아이디)와 비밀번호를 입력해 로그인합니다.

- **화면 구성**  
  - 전체 배경에 설정된 이미지  
  - 카드 형태 영역 안:  
    - 앱 로고("MEMO-RE")  
    - 이메일 입력란
    - 비밀번호 입력란
    - “로그인” 버튼  
    - “회원가입 | 비밀번호 찾기” 

- **링크 동작**  
  - “회원가입”을 누르면 SignupScreen으로 이동  
  - “비밀번호 찾기”를 누르면 ForgotPasswordScreen으로 이동

---

### 2. 회원가입 화면 (SignupScreen)
- **무엇을 하나요?**  
  사용자는 이메일, 비밀번호, 비밀번호 확인, 닉네임을 입력해 회원가입을 합니다.  
- **어떻게 동작하나요?**  
  1. 네 개의 입력란(이메일·비밀번호·비밀번호 확인·닉네임)을 모두 채워야 합니다.  
  2. 비밀번호와 비밀번호 확인이 다르면 “비밀번호가 일치하지 않습니다”라는 경고가 나타납니다.  
  3. 입력이 모두 올바르면 서버로 회원가입 요청을 보내고, 성공하면 자동으로 로그인합니다.  
  4. 로그인에 성공하면 사용자 토큰과 닉네임이 기기에 저장되며, 보드 목록 화면으로 이동합니다.  
- **사용자가 보는 화면**  
  - 상단에 “회원가입” 제목  
  - 이메일, 비밀번호, 비밀번호 확인, 닉네임을 입력하는 네 개의 입력란  
  - “가입하기” 버튼  
  - “뒤로가기” 버튼(이전 화면으로 돌아가기)  

---

### 3. 비밀번호 재설정 화면 (ForgotPasswordScreen)
- **무엇을 하나요?**  
  가입한 이메일 주소와 새 비밀번호를 입력하면, 서버에 비밀번호 재설정 요청을 보냅니다.

- **어떻게 동작하나요?**  
  1. 이메일과 새 비밀번호 입력란에 값을 모두 채워야 합니다.  
  2. “비밀번호 재설정” 버튼을 누르면 다음 API 요청이 전송됩니다:
     ```
     POST /api/reset-password/
     { email: "<가입한 이메일>", new_password: "<새 비밀번호>" }
     ```
  3. 서버가 성공 응답(200)을 반환하면 “비밀번호가 재설정되었습니다”라는 알림을 띄우고 로그인 화면으로 이동합니다.  
  4. 오류가 발생하면, 서버 에러 메시지를 Alert로 보여줍니다.  
  5. 하단의 “로그인 화면으로 돌아가기” 링크를 누르면 로그인 화면으로 돌아갑니다.

- **사용자가 보는 화면 구성**  
  - 상단: “비밀번호 재설정” 제목  
  - 중간:  
    - “가입한 이메일 주소” 입력란  
    - “새로운 비밀번호” 입력란  
    - “비밀번호 재설정” 버튼  
  - 하단: “로그인 화면으로 돌아가기” 링크 (로그인 화면으로 네비게이션)

---

### 4. 보드 목록 화면 (BoardListScreen)
- **무엇을 하나요?**  
  사용자 본인이 만든 보드(아이디어 주제)와 이웃(공유) 보드를 한 화면에서 보여주고, 새 보드를 만들거나 이웃을 관리할 수 있습니다.

- **어떻게 동작하나요?**  
  1. **초기화 시**:  
     - 서버에서 내 보드 목록, 이웃 목록, 이웃이 공유한 보드·메모, 받은 이웃 요청을 동시에 불러옵니다.
     - “이웃의 보드 목록” 섹션에는 `{username}`이 공유한 보드 제목을 나열합니다.
  2. **내 보드 목록**:  
     - 내 보드를 탭하면 해당 보드의 메모 화면으로 이동합니다.
     - 화면 하단의 입력란에 보드 이름을 입력 후 “보드 추가” 버튼을 누르면, 서버에 새 보드가 생성되고 목록이 갱신됩니다.
  3. **이웃의 보드 목록**:  
     - 이웃이 공유한 보드를 탭하면, 해당 보드 안의 메모만 따로 모아서 미리보기 형태로 보여줍니다.
     - 안내용 첫 항목(“보드 이름을 적고 보드 추가를 눌러주세요”)을 누르면 안내 보드(사용법 안내)로 이동합니다.
  4. **이웃 관리 모달**:  
     - 헤더의 “이웃 관리”를 누르면 모달이 열립니다.
     - **검색 탭**: 사용자 이름을 입력해 이웃 요청을 보낼 수 있습니다.
     - **받은 이웃 요청 탭**: 다른 사람이 보낸 이웃 요청을 “수락” 또는 “거절”할 수 있습니다.
     - **내 이웃 목록 탭**: 현재 내 이웃(팔로잉) 목록이 나오며, 옆의 “이웃 취소” 버튼으로 이웃을 삭제할 수 있습니다.
     - 모달 하단의 “닫기” 버튼을 누르면 이웃 관리 화면이 닫힙니다.

---

### 5. 메모보드 화면 (MemoBoardScreen)
- **무엇을 하나요?**  
  특정 보드에 속한 메모들을 모아서 보여주고, 새 메모를 추가하거나 전체 내용을 요약할 수 있는 화면입니다.  
- **어떻게 동작하나요?**  
  1. **보드 제목 표시**  
     - 이전 화면(보드 목록)에서 전달된 보드 제목이 있으면 바로 보여줍니다.  
     - 없다면 서버에서 실제 보드 제목을 받아와서 화면 상단에 표시합니다.  
  2. **메모 리스트 보여주기**  
     - 서버에서 해당 보드에 작성된 메모 전체를 가져와 날짜·내용·시간 순서로 화면에 나열합니다.  
     - “사용법 안내 보드”(특별한 안내용 보드)의 경우, 미리 정의된 안내 문구 한 줄만 표시합니다.  
  3. **새 메모 추가**  
     - 화면 하단 입력란에 글을 쓰고 “메모 추가” 버튼을 누르면, 즉시 목록에 추가됩니다.  
     - “사용법 안내 보드”일 때는 메모 추가가 불가능하며, 버튼이 비활성화됩니다.  
  4. **전체 요약 기능**  
     - “정리하기” 버튼을 누르면, 현재 보드에 속한 모든 메모를 한 번에 요약해서 보여줍니다.  
     - “사용법 안내 보드”일 때는 요약 기능을 사용할 수 없습니다.  
- **사용자가 보는 화면**  
  - 상단: 뒤로 가기 화살표(←)와 보드 제목(📝 제목)  
  - 중간:  
    - 메모가 여러 개 있으면 날짜·내용·시간 순서대로 세로 목록(FlatList) 형태로 보여줌  
    - 요약 결과가 있으면 “📌 전체 요약:” 레이블과 요약 텍스트 박스  
  - 하단:  
    - 입력란(새 메모 입력)  
    - “메모 추가” 버튼  
    - “정리하기” 버튼

  5. **로그아웃**:  
     - 화면 상단 우측의 “로그아웃” 버튼을 누르면 저장된 토큰과 사용자 정보가 삭제되고 로그인 화면으로 돌아갑니다.

- **사용자가 보는 화면 구성**  
  - 상단: “📋 메모 보드 목록” 타이틀, “이웃 관리” 버튼, “로그아웃” 버튼  
  - 중간:  
    - “내 보드 목록” 섹션: 내 보드 제목들(FlatList)  
    - “이웃의 보드 목록” 섹션: 이웃이 공유한 보드 제목들(FlatList)  
  - 하단:  
    - 보드 이름 입력란  
    - “보드 추가” 버튼  
  - **이웃 관리 모달** (화면 중앙에 오버레이)  
    - 탭 구분 없이 세로로 나열된 섹션: 검색, 받은 요청, 내 이웃  
    - 각 섹션 옆에 “이웃 요청”, “수락/거절”, “이웃 취소” 버튼  
    - “닫기” 버튼
